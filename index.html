What is EVE ?

1001 Flavors of SIMD

  • X86
    • 128 bits: SSE2,SSSE3,SSE3, SSE4.1, SSE4.2
    • 256 bits: AVX, AVX2, XOP
    • 512 bits: AVX512 and its myriad of sub-genre
  • ARM
    • 128 bits: NEON, ASIMD
  • PowerPC
    • 128 bits: Altivec on Power7-9
    • 256 bits: Blue Gene Q QPX

How to exploit SIMD ?

  • Compiler's autovectorization
  • Pragmas/special compilers
  • std::execution::unseq
  • Specialized tools: Halide, simdjson
  • write by hand

EVE - The Expressive Velocity Engine

  • C++ 20 wrapper around SIMD extensions
    • Type wrappers
    • Algorithms : complete and building blocks
    • 250+ numerical functions
  • Supports all X86 flavors and AARCH64

EVE Type Wrappers

  • eve::wide<Type, eve::fixed<Lanes>>
    • Contains Lanes elements of type Type
    • Type is either an arithmetic type or a product type
    • Lanes is any power of two
  • eve::logical<Type>
    • Represents a logical value
    • Works for both scalar and wide types
    • Enables optimizations

EVE as a C++20 library

Objectives

  • Overload over architecture, instruction sets, types
  • Adding specific optimization must be allowed
  • Easy-to-use, Easy-to-discover API

Initial Design

  • 50% functions, 50% functions calling Callable Object internally
  • Used tag dispatching and SFINAE
  • Resolving multiple overloads was common
  • Compile at 11, get lunch, compilation is done for coffee

C++17/20 Design Decision

  • Detect SIMD architectures and instructions sets via traits
  • Use Concepts to overload on SIMD architecture
  • Use if constexpr to write code based on available SIMD instructions sets
  • Use enum based categorization to simplify type recognition
